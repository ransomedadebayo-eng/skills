# MVP/Product Discovery Partner

## Overview

This skill helps you validate and structure product ideas through systematic discovery—defining problems, understanding users, prioritizing features, and planning lean MVPs before you write a single line of code.

**Keywords**: MVP, product discovery, lean startup, user personas, product requirements, feature prioritization, validation experiments, product-market fit

## When to Use This Skill

- Validate new product ideas
- Define MVP scope
- Create user personas
- Prioritize features
- Write PRDs (Product Requirements Documents)
- Plan validation experiments
- Structure SaaS or app projects

```

[Product Name] - MVP
Version: 1.0
Date: [Date]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. OVERVIEW
[1-paragraph description of the product]

2. PROBLEM & SOLUTION
Problem: [Problem statement]
Solution: [How this product solves it]
Target User: [Who it's for]

3. USER STORIES

As a [user type],
I want to [action],
So that [benefit].

[Repeat for 5-7 core user stories]

4. FEATURE SPEC

Feature 1: [Name]
- Description: [What it does]
- User flow: [Step-by-step]
- Acceptance criteria:
  □ [Criterion 1]
  □ [Criterion 2]

[Repeat for all MVP features]

5. OUT OF SCOPE
- [What we're NOT building in V1]

6. SUCCESS METRICS
- [Metric 1]: [Target]
- [Metric 2]: [Target]

7. TECHNICAL CONSIDERATIONS
- Platform: [Web/Mobile/Both]
- Tech Stack: [Recommendations]
- Integrations: [Third-party services]
- Hosting: [Where it will run]
```

## MVP Roadmap

```
MVP ROADMAP: [Product Name]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

PHASE 1: DISCOVERY (Weeks 1-2)
□ User interviews (10-15 people)
□ Define problem statement
□ Create personas
□ Map user journeys

PHASE 2: VALIDATION (Weeks 3-4)
□ Create landing page
□ Run validation experiment
□ Analyze results
□ Refine concept

PHASE 3: DESIGN (Weeks 5-6)
□ Create wireframes
□ User flow diagrams
□ Basic UI mockups
□ Feedback from potential users

PHASE 4: BUILD (Weeks 7-12)
□ Set up infrastructure
□ Build core features
□ Internal testing
□ Bug fixes

PHASE 5: BETA (Weeks 13-14)
□ Launch to beta users (20-50)
□ Gather feedback
□ Iterate based on feedback
□ Monitor metrics

PHASE 6: LAUNCH (Week 15+)
□ Public launch
□ Monitor key metrics
□ User support
□ Plan V2 features
```

## Example: CRM for Pharmacy

```
PRODUCT CONCEPT: PharmaCRM
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

PROBLEM
Small pharmacy owners struggle to stay in touch with customers for refills and health check-ins, leading to lost revenue and lower customer retention.

SOLUTION
Simple CRM to track customer refill schedules, send automated reminders, and log interactions.

PERSONA: "Sarah the Pharmacy Owner"
Age: 45
Pain: Manually tracking refills in notebook, forgetting to follow up
Goal: Increase refill adherence and customer loyalty

MVP FEATURES (Must Have)
□ Customer database (name, phone, meds, refill schedule)
□ Automated refill reminders (SMS/Email)
□ Simple dashboard showing upcoming refills
□ Interaction log (when you last contacted them)

VALIDATION EXPERIMENT
Landing page describing the product
Target: 20 pharmacy owners sign up for waitlist
Method: LinkedIn outreach to pharmacy groups

TECH STACK
- Frontend: React
- Backend: Firebase/Supabase
- SMS: Twilio
- Hosting: Vercel

SUCCESS METRICS
- 50% of customers respond to reminders
- 20% increase in refill adherence
- Pharmacy owner uses it 3x/week

BUILD TIME: 6-8 weeks
```
